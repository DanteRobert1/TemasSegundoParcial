CONTENEDORES

QUE ES Y SU USO

LOS CONTENEDORES SON PAQUETES DE SOFTWARE QUE INCLUYEN TODOS LOS ELEMENTOS NECESARIOS PARA EJECUTAR APLICACIONES EN CUALQUIER ENTORNO. VIRTUALIZAN EL SISTEMA OPERATIVO, LO QUE LES PERMITE EJECUTARSE EN DIFERENTES LUGARES, COMO CENTROS DE DATOS PRIVADOS, NUBES PÚBLICAS O INCLUSO EN EL PORTÁTIL PERSONAL DE UN DESARROLLADOR.

  • LOS CONTENEDORES SE PUEDEN EJECUTAR EN CUALQUIER SISTEMA QUE ADMITA SU TECNOLOGÍA DE VIRTUALIZACIÓN A NIVEL DE SISTEMA OPERATIVO. ESTO INCLUYE SERVIDORES FÍSICOS, MÁQUINAS VIRTUALES, NUBES PÚBLICAS Y PRIVADAS, ASÍ COMO ESTACIONES DE TRABAJO DE DESARROLLO.

    • VENTAJAS

    • SEPARACIÓN DE RESPONSABILIDADES: YA QUE LOS DESARROLLADORES SE CENTRAN EN LA LÓGICA Y LAS DEPENDENCIAS DE LAS APLICACIONES.
    • PORTABILIDAD DE LAS CARGAS DE TRABAJO: COMO LOS CONTENEDORES SE PUEDEN EJECUTAR CASI EN CUALQUIER LUGAR, RESULTA FACILÍSIMO DESARROLLARLOS Y DESPLEGARLOS EN SISTEMAS OPERATIVOS LINUX, WINDOWS Y MAC.
    • AISLAMIENTO DE APLICACIONES: LOS CONTENEDORES VIRTUALIZAN LOS RECURSOS DE CPU, MEMORIA, ALMACENAMIENTO Y RED A NIVEL DE SISTEMA OPERATIVO, LO QUE PROPORCIONA A LOS DESARROLLADORES UNA VISTA DEL SISTEMA OPERATIVO AISLADO.

    MAQUINA VIRTUAL VS CONTENEDORES

    ASPECTO MAQUINA VIRTUAL CONTENEDOR
    PESO MÁS PESADAS,REQUIEREN MÁS RECURSOS. MUCHO MÁS LIGERAS, UTILIZAN MENOS RECURSOS.
    NIVEL DE VIRTUALIZACIÓN VIRTUALIZAN A NIVEL DE HARDWARE COMPLETO. VIRTUALIZAN A NIVEL DE SISTEMA OPERATIVO.
    USO DE MEMORIA UTILIZAN MÁS MEMORIA DEBIDO AL SISTEMA OPERATIVO COMPLETO. UTILIZAN MENOS MEMORIA AL COMPARTIR EL KERNEL DEL SISTEMA OPERATIVO.
    AISLAMIENTO AÍSLAN POR COMPLETO CADA INSTANCIA CON SU PROPIO SISTEMA OPERATIVO. AÍSLAN APLICACIONES PERO COMPARTEN EL KERNEL DEL SISTEMA OPERATIVO.
    ARRANQUE Y EJECUCIÓN MÁS LENTO DEBIDO AL INICIO DEL SISTEMA OPERATIVO COMPLETO. MÁS RÁPIDO DEBIDO A LA COMPARTICIÓN DEL KERNEL Y MENOS CAPAS DE VIRTUALIZACIÓN.
