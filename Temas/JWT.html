Tecnologías de Web API

JWT

JWT QUE ES?

ES UN ESTÁNDAR PARA LA AUTENTICACIÓN Y EL INTERCAMBIO DE INFORMACIÓN. SI UN JWT ESTÁ FIRMADO POR UN PAR DE CLAVES PÚBLICA/PRIVADA, LA EMPRESA CERTIFICA QUE LA PARTE QUE TIENE LA CLAVE PRIVADA ESTÁ REALMENTE FIRMADA.

TOKEN

ESTOS DATOS SE CONVIERTEN EN UNA FIRMA DIGITAL QUE TE IDENTIFICARÁ DE MANERA ÚNICA PARA ESE BANCO Y CADA VEZ QUE ACCEDAS A TU BANCO E INGRESES TU CONTRASEÑA Y DATOS PERSONALES, EL BANCO COMPRENDERÁ Y CONFIRMARÁ QUE TÚ ERES EL USUARIO QUE INICIÓ LA SESIÓN.

COMPONENTES

HEADER

HEADERS SON LOS ENCABEZADOS DEL TOKEN DONDE PASAMOS BÁSICAMENTE DOS PIEZAS DE INFORMACIÓN: EL ALGORITMO QUE INFORMA QUÉ ALGORITMO SE USA PARA CREAR LA FIRMA Y EL TYP QUE INDICA EL TIPO DE TOKEN.

PAYLOAD

EL PAYLOAD ES EL COMPONENTE DONDE SE PUEDEN ENCONTRAR DATOS DE AUTENTICACIÓN COMO CONTRASEÑAS Y CORREOS ELECTRÓNICOS.

SIGNATURE

LA FIRMA DEL TOKEN (FIRMA) ESTÁ COMPUESTA POR LA CODIFICACIÓN DEL ENCABEZADO Y EL PAYLOAD MÁS UNA CLAVE SECRETA Y ES GENERADA POR EL ALGORITMO ESPECIFICADO EN EL ENCABEZADO.
