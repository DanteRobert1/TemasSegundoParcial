¿QUE ES?

MARCO DE PRESENTACIÓN LIGERO Y MUY FÁCIL DE PROBAR QUE (COMO LAS APLICACIONES BASADAS EN WEB FORMS) SE INTEGRA CON LAS CARACTERÍSTICAS DE ASP.NET EXISTENTES, TALES COMO PÁGINAS MAESTRAS Y LA AUTENTICACIÓN BASADA EN PERTENENCIA. PROPORCIONA UNA ALTERNATIVA AL PATRÓN DE ASP.NET WEB FORMS PARA CREAR APLICACIONES WEB BASADAS EN MVC.

OFRECE

HERRAMIENTA O PLATAFORMA QUE CUMPLE CON TUS REQUISITOS ES ANGULAR. ANGULAR ES UN FRAMEWORK DE DESARROLLO DE APLICACIONES WEB DESARROLLADO Y MANTENIDO POR GOOGLE. OFRECE UN ENFOQUE EN LA SEPARACIÓN DE PREOCUPACIONES (COMO EL PATRÓN MVC), UNA AMPLIA FUNCIONALIDAD TANTO EN EL LADO DEL CLIENTE COMO DEL SERVIDOR, Y SOPORTE PARA DIVERSAS NECESIDADES DE DESARROLLO Y DESPLIEGUE.

CARACTERISTICAS

  • LA SEPARACIÓN DE TAREAS EN ASP.NET PERMITE DIVIDIR LA LÓGICA DE ENTRADA, LÓGICA EMPRESARIAL Y LÓGICA DE LA INTERFAZ DE USUARIO, LO QUE FACILITA LAS PRUEBAS Y EL DESARROLLO BASADO EN PRUEBAS (TDD) DE FORMA PREDETERMINADA.
  • EL MARCO DE MVC DE ASP.NET ES EXTENSIBLE Y CONECTABLE, LO QUE PERMITE REEMPLAZAR O PERSONALIZAR FÁCILMENTE SUS COMPONENTES. SE PUEDE CONECTAR UN MOTOR DE VISTA PERSONALIZADO, DIRECTIVAS DE ENRUTAMIENTO DE URL, SERIALIZACIÓN DE PARÁMETROS DE MÉTODO Y ACCIÓN, ENTRE OTROS COMPONENTES.
  • TIENE UN COMPONENTE DE MAPEO DE URL EFICAZ QUE FACILITA LA CREACIÓN DE APLICACIONES CON DIRECCIONES URL COMPRESIBLES Y FÁCILES DE BUSCAR.
  • SOPORTE TÉCNICO PARA UTILIZAR EL MARCADO EN ARCHIVOS DE PÁGINAS EXISTENTES DE ASP.NET (ARCHIVOS .ASPX), CONTROLES DE USUARIO (ARCHIVOS .ASCX) Y PÁGINAS MAESTRAS (ARCHIVOS .MASTER) COMO PLANTILLAS DE VISTA.
  • PERMITE USAR CARACTERÍSTICAS, TALES COMO LA AUTENTICACIÓN MEDIANTE FORMULARIOS Y LA AUTENTICACIÓN DE WINDOWS, LA AUTORIZACIÓN PARA DIRECCIONES URL, LA PERTENENCIA Y LOS ROLES, EL ALMACENAMIENTO EN CACHÉ DE RESULTADOS Y DATOS.
