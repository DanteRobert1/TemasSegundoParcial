WCF

WINDOWS COMMUNICATION FOUNDATION

ES UN MARCO PARA CREAR APLICACIONES ORIENTADAS A SERVICIOS. CON WCF, ES POSIBLE ENVIAR DATOS COMO MENSAJES ASINCRÓNICOS DE UN PUNTO DE CONEXIÓN DE SERVICIO A OTRO. UN EXTREMO PUEDE SER UN CLIENTE DE UN SERVICIO QUE SOLICITA DATOS DE UN EXTREMO DE SERVICIO. LOS MENSAJES PUEDEN SER TAN SIMPLES COMO UN CARÁCTER O UNA PALABRA QUE SE ENVÍA COMO XML, O TAN COMPLEJOS COMO UNA SECUENCIA DE DATOS BINARIOS.

CARACTERISTICAS

ORIENTACION A SERVICIOS

USO DE LOS ESTÁNDARES DE WS ES QUE WCF LE PERMITE CREAR APLICACIONES ORIENTADAS A SERVICIOS. SOA, LA ARQUITECTURA ORIENTADA A SERVICIOS, ES EL USO DE SERVICIOS WEB PARA ENVIAR Y RECIBIR DATOS. LOS SERVICIOS TIENEN LA VENTAJA GENERAL DE ESTAR DÉBILMENTE ACOPLADOS ENTRE UNA APLICACIÓN Y OTRA EN LUGAR DE ESTAR INCLUIDOS EN EL CÓDIGO.

INTEROPELABILIDAD

WCF IMPLEMENTA ESTÁNDARES DEL SECTOR MODERNOS PARA LA INTEROPERABILIDAD DE LOS SERVICIOS WEB.

METADATOS DE SERVICIOS

WCF ADMITE LA PUBLICACIÓN DE METADATOS DE SERVICIOS CON LOS FORMATOS ESPECIFICADOS EN LOS ESTÁNDARES DE LA INDUSTRIA, COMO WSDL, ESQUEMA XML Y WS-POLICY. ESTOS METADATOS PUEDEN UTILIZARSE PARA GENERAR Y CONFIGURAR CLIENTES DE FORMA AUTOMÁTICA PARA EL ACCESO A LOS SERVICIOS DE WCF. LOS METADATOS SE PUEDEN PUBLICAR SOBRE HTTP Y HTTPS.

SEGURIDAD

ES POSIBLE CIFRAR LOS MENSAJES PARA PROTEGER LA PRIVACIDAD, ASÍ COMO OBLIGAR A LOS USUARIOS A QUE SE AUTENTIQUEN ANTES DE PERMITIRLES RECIBIR MENSAJES. LA SEGURIDAD PUEDE IMPLEMENTARSE UTILIZANDO ESTÁNDARES CONOCIDOS COMO SSL O WS-SECURECONVERSATION.

MENSAJES CONFIABLES Y EN COLA

WCF ADMITE EL INTERCAMBIO DE MENSAJES CONFIABLE MEDIANTE SESIONES CONFIABLES IMPLEMENTADAS A TRAVÉS DE MENSAJES DE CONFIANZA DE WS Y CON MSMQ.

MENSAJES DURADEROS

UN MENSAJE DURADERO ES AQUEL QUE NUNCA SE PIERDE DEBIDO A UNA INTERRUPCIÓN DE LA COMUNICACIÓN. LOS MENSAJES QUE FORMAN PARTE DE UN PATRÓN DE MENSAJES DURADEROS SIEMPRE SE GUARDAN EN UNA BASE DE DATOS. SI SE PRODUCE UNA INTERRUPCIÓN, LA BASE DE DATOS LE PERMITE REANUDAR EL INTERCAMBIO DE MENSAJES CUANDO SE RESTABLEZCA LA CONEXIÓN.

EXTENSIBILIDAD

LA ARQUITECTURA DE WCF TIENE VARIOS PUNTOS DE EXTENSIBILIDAD. SI SE NECESITA UNA FUNCIÓN ADICIONAL, EXISTEN UNA SERIE DE PUNTOS DE ENTRADA QUE LE PERMITEN PERSONALIZAR EL COMPORTAMIENTO DE UN SERVICIO.

PASOS PARA CREAR UN WCF

WCF

  1. CREAR UN PROYECTO LLAMADO APLICACIÓN DE SERVICIOS WCF
  2. CAMBIAR EL NOMBRE DEL ARCHIVO Y DE LA CLASE
  3. BORRAR MÉTODOS DE EJEMPLOS QUE NO NOS INTERESAN
  4. AGREGAR UNA BASE DE DATOS
  5. IR A LAS TABLAS Y UNA VEZ METEMOS LOS CAMPOS ACTUALIZAR PARA QUE APAREZCA LA TABLA
  6. AGREGAR CARPETA Y EN ESTA AGREGAR NUEVO ELEMENTO LLAMADO ADO.NET ENTITY
  7. PONER DESIGNED DE BASE DE DATOS Y EN NUEVA CONEXIÓN
  8. VINCULAR CON LAS TABLAS
  9. DEFINIR EL MÉTODO EN LA MAIN
  10. PONER EL USING PARÉNTESIS BASE DE DATOS CONTEXTO = NUW BASE DE DATOS
  11. RETURN CONTEXTO.CLASE.ASNOTTRACKING().LIST()
  12. DENTRO DE LA SOLUCIÓN AGREGAR NUEVO PROYECTO Y EN APLICACIÓN WEB ASP.NET Y CREAR WEB FORMS
  13. EN REFERENCIAS AGREGAR REFERENCIAS DE SERVICIO
  14. EN WEB FORMS AGREGAR FORMULARIO WEB FORMS LLAMADO GRILLA
  15. AHÍ PUEDES PONER EL DISEÑO QUE TU QUIERAS
  16. AL QUERER CONSULTAR UTILIZAS UN MÉTODO CON USING CON UNA REFERENCIA A TU ARCHIVO CON TU SERVICIO
  17. AL PONER EL GRILLA PARA TENER EL MÉTODO PARA QUE EL MÉTODO DEL ORIGINAL SEA REFLEJADO
  18. UNA VEZ QUE LOS DATOS ESTÉN SOLO PONER GRILLA.DATABIND
  19. EN UN BOTÓN PONES LA CONSULTA PARA QUE AL MOMENTO QUE SE PULSE SE MUESTRE
  20. UNA VEZ QUE SE MUESTRE LA PÁGINA DEBERÍA FUNCIONAR
